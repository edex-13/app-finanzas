Ya tengo funcionando autenticación, onboarding, dashboard, cuentas, tarjetas y deudas conectadas a Supabase.

Ahora necesito construir el módulo de transacciones reales.

Objetivo:
Permitir que el usuario registre ingresos, gastos, pagos de deuda, pagos de tarjeta, transferencias y ajustes, afectando correctamente su estado financiero.

Tareas:

1. Crear pantalla de transacciones:

* Listado de transacciones.
* Filtros por fecha.
* Filtros por tipo.
* Filtros por categoría.
* Filtros por cuenta.
* Filtros por tarjeta.
* Buscador por descripción.
* Botón para crear nueva transacción.

2. Crear formulario de transacción con estos tipos:

* Ingreso
* Gasto
* Pago de deuda
* Pago de tarjeta
* Transferencia
* Ajuste

3. Campos generales:

* Tipo
* Valor
* Fecha
* Descripción
* Categoría
* Cuenta asociada
* Tarjeta asociada
* Deuda asociada

4. Reglas para ingresos:

* Deben aumentar el saldo de una cuenta.
* Deben poder asociarse a una categoría de ingreso.
* Ejemplo: salario, pago extra, devolución.

5. Reglas para gastos:

* Si se paga con cuenta, debe disminuir el saldo de esa cuenta.
* Si se paga con tarjeta, debe aumentar la deuda de la tarjeta.
* Si se paga con tarjeta, preguntar:

  * Número de cuotas.
  * Tiene intereses: sí/no.
  * Tasa de interés si aplica.
  * Fecha de compra.
* Si es a cuotas, debe crear deuda o cuotas futuras asociadas.

6. Reglas para pago de deuda:

* Debe disminuir el saldo de la cuenta usada para pagar.
* Debe disminuir el saldo pendiente de la deuda.
* Debe actualizar cuotas si aplica.

7. Reglas para pago de tarjeta:

* Debe disminuir el saldo de la cuenta usada para pagar.
* Debe disminuir la deuda actual de la tarjeta.
* No debe permitir pagar más que la deuda actual.

8. Reglas para transferencia:

* Debe mover dinero de una cuenta origen a una cuenta destino.
* No debe cambiar el patrimonio total.
* Debe validar saldo suficiente.

9. Reglas para ajuste:

* Permite corregir saldos manualmente.
* Debe guardar descripción obligatoria.
* Debe dejar trazabilidad.

10. Crear funciones puras:

* applyIncomeTransaction
* applyExpenseTransaction
* applyDebtPaymentTransaction
* applyCreditCardPaymentTransaction
* applyTransferTransaction
* applyAdjustmentTransaction
* validateSufficientBalance
* calculateTransactionImpact

11. Crear hooks:

* useTransactions
* useCreateTransaction
* useUpdateTransaction
* useDeleteTransaction
* useTransactionFilters

12. Importante:
    Las operaciones que afectan varias tablas deben hacerse de forma segura.
    Si Supabase RPC es necesario, crear funciones SQL para registrar transacciones de manera atómica.

13. Crear una función RPC en Supabase llamada create_financial_transaction si es necesario.
    Debe:

* Insertar la transacción.
* Actualizar saldo de cuenta.
* Actualizar deuda de tarjeta o deuda general.
* Mantener consistencia.

14. UI:

* Tabla clara de transacciones.
* Badges por tipo.
* Valor en verde para ingresos.
* Valor en rojo para gastos.
* Detalle de impacto financiero.
* Confirmación antes de eliminar.

15. Entregable:

* Código completo.
* SQL adicional si se necesita RPC.
* Hooks.
* Componentes.
* Validaciones.
* Funciones financieras.
* Explicación de cómo probar:

  * ingreso
  * gasto con cuenta
  * gasto con tarjeta
  * pago de deuda
  * pago de tarjeta
  * transferencia
  * ajuste
