Ya tengo autenticación, onboarding y dashboard básico conectados a Supabase.

Ahora necesito mejorar los módulos principales de configuración financiera:

* Cuentas
* Tarjetas de crédito
* Deudas

Objetivo:
Crear módulos CRUD completos para que el usuario pueda administrar su estado financiero real después del onboarding.

Tareas:

1. Módulo de cuentas:
   Crear una pantalla para listar, crear, editar y eliminar cuentas.

Campos:

* Nombre
* Tipo: efectivo, banco, billetera digital, otro
* Saldo actual
* Moneda
* Notas opcionales

Reglas:

* Cada cuenta pertenece al usuario autenticado.
* Al crear una transacción de gasto desde una cuenta, el saldo debe poder actualizarse correctamente más adelante.
* Mostrar saldo total de todas las cuentas.

2. Módulo de tarjetas de crédito:
   Crear una pantalla para listar, crear, editar y eliminar tarjetas.

Campos:

* Nombre de la tarjeta
* Banco
* Cupo total
* Deuda actual
* Cupo disponible calculado
* Día de corte
* Día máximo de pago
* Color visual
* Notas

Reglas:

* El cupo disponible debe calcularse como cupo total menos deuda actual.
* Validar que fecha de corte sea entre 1 y 31.
* Validar que fecha de pago sea entre 1 y 31.
* Mostrar visualmente si la tarjeta tiene alto nivel de uso.
* Crear función calculateCreditCardUtilization.

3. Módulo de deudas:
   Crear una pantalla para listar, crear, editar y eliminar deudas.

Campos:

* Nombre
* Tipo de deuda
* Valor total
* Saldo pendiente
* Tiene intereses: sí/no
* Tasa de interés
* Frecuencia de pago
* Próxima fecha de pago
* Número total de cuotas
* Cuotas restantes
* Valor de cuota
* Medio de pago
* Notas

Reglas:

* Si la deuda tiene cuotas, permitir generar cuotas futuras.
* Si tiene intereses, dejar campo de tasa.
* Si no tiene intereses, la tasa debe ser 0.
* Calcular progreso de pago.
* Mostrar cuotas restantes.

4. Crear formularios con:

* React Hook Form
* Zod
* Validaciones claras
* Estados de carga
* Estados de error
* Confirmación antes de eliminar

5. Crear hooks:

* useAccounts
* useCreateAccount
* useUpdateAccount
* useDeleteAccount
* useCreditCards
* useCreateCreditCard
* useUpdateCreditCard
* useDeleteCreditCard
* useDebts
* useCreateDebt
* useUpdateDebt
* useDeleteDebt

6. Usar TanStack Query para:

* cachear datos
* invalidar consultas al crear, editar o eliminar
* mostrar loading states

7. UI:

* Usar cards y tablas limpias.
* Diseño moderno y minimalista.
* Botones claros.
* Formularios en modal o páginas dedicadas.
* Estados vacíos útiles.

8. Seguridad:

* Todas las operaciones deben usar el usuario autenticado.
* No permitir crear registros sin user_id.
* Mantener compatibilidad con las políticas RLS de Supabase.

Entregable:

* Código completo de los archivos creados o modificados.
* Componentes.
* Hooks.
* Validaciones.
* Tipos.
* Funciones de cálculo.
* Instrucciones para probar cada módulo.
